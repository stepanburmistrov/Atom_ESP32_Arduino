# UART-протокол RobotX

## Назначение

UART-протокол нужен, чтобы разные пульты управления могли одинаково управлять Arduino Nano:

- Python-пульт на компьютере отправляет команды через USB Serial Arduino Nano;
- ESP32-C3 получает команды с веб-страницы и пересылает их в Arduino Nano через `Serial1`;
- Arduino Nano не знает, кто именно отправил команду, он просто принимает символы по UART.

## Параметры UART

| Параметр | Значение |
|---|---|
| Скорость | `9600` бод |
| Формат | `SERIAL_8N1` |
| Окончание команды | `
` |
| Кодировка | ASCII |
| Длина полезной команды | 1 символ |

Пример фактической строки:

```text
F

```

## Команды движения

| Команда | Действие Arduino Nano |
|---|---|
| `F` | оба мотора вперёд |
| `B` | оба мотора назад |
| `L` | левый мотор назад, правый вперёд |
| `R` | левый мотор вперёд, правый назад |
| `S` | оба мотора остановлены |

В Python-пульте и веб-пульте команды движения работают как **удерживаемые кнопки**: при нажатии отправляется `F/B/L/R`, при отпускании — `S`.

## Команды сервоприводов

| Команда | Действие |
|---|---|
| `Q` | начать движение левой сервы вверх |
| `q` | остановить движение левой сервы вверх |
| `A` | начать движение левой сервы вниз |
| `a` | остановить движение левой сервы вниз |
| `W` | начать движение правой сервы вверх |
| `w` | остановить движение правой сервы вверх |
| `D` | начать движение правой сервы вниз |
| `d` | остановить движение правой сервы вниз |

Правая серва установлена зеркально, поэтому внутри Arduino-скетча направления `W/D` специально инвертированы.

## Почему команды верхнего и нижнего регистра разные

Для сервоприводов верхний регистр означает **начать движение**, нижний регистр — **остановить соответствующее движение**.

Например:

```text
Q
  левая серва начала двигаться вверх
q
  левая серва остановилась
```

Это удобно для кнопок на экране и клавиатуре: кнопка нажата — серва едет, кнопка отпущена — серва остановилась.

## Проверка без робота

В Arduino IDE можно открыть Serial Monitor на `9600` бод и отправлять команды вручную:

```text
F
S
Q
q
A
a
```

Если включён `DEBUG_SERIAL`, Nano будет печатать диагностические сообщения. При работе с ESP32 лучше отключить отладку:

```cpp
#define DEBUG_SERIAL 0
```
