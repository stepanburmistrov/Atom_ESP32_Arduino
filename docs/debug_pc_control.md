# Python-пульт для отладки с компьютера

Файл: `pc_control/robotx_uart_control.py`

![Python UART пульт](../images/pc_uart_control_screenshot.png)

## Зачем нужен

Этот пульт нужен, чтобы отлаживать Arduino Nano и механику робота **до подключения ESP32**.

Схема работы:

```text
Компьютер
  ↓ USB кабель
Arduino Nano
  ↓
моторы и сервоприводы
```

Python отправляет в Arduino Nano те же команды, которые позже будет отправлять ESP32.

## Установка

В папке репозитория выполнить:

```bash
pip install -r pc_control/requirements.txt
```

## Запуск

```bash
python pc_control/robotx_uart_control.py
```

## Управление мышью

Кнопки на экране работают как удерживаемые:

- нажали кнопку — команда отправилась;
- отпустили кнопку — отправилась команда остановки.

Для движения при отпускании отправляется `S`.
Для сервоприводов при отпускании отправляется соответствующая маленькая буква: `q`, `a`, `w`, `d`.

## Управление клавиатурой

| Клавиша | Команда |
|---|---|
| `↑` | `F` — вперёд |
| `↓` | `B` — назад |
| `←` | `L` — влево |
| `→` | `R` — вправо |
| `Space` | `S` — стоп |
| `Q` | `Q/q` — левая серва вверх / стоп |
| `A` | `A/a` — левая серва вниз / стоп |
| `W` | `W/w` — правая серва вверх / стоп |
| `D` | `D/d` — правая серва вниз / стоп |

Для моторов специально используются стрелки, чтобы не конфликтовать с командами сервоприводов `Q/W/A/D`.

## Настройки

По умолчанию скорость:

```text
9600 бод
```

Она должна совпадать с Arduino:

```cpp
const int UART_BAUD = 9600;
```

## Режим DEMO

Если COM-порт не подключён, кнопки всё равно будут писать команды в лог как `DEMO TX`. Это удобно, чтобы проверить интерфейс без робота.

## Частые проблемы

### COM-порт не появился

Проверьте:

- подключён ли USB кабель;
- установлен ли драйвер CH340/FTDI;
- не открыт ли этот порт в Arduino IDE Serial Monitor.

### Arduino перезагружается при подключении

Это нормально: многие платы Nano перезапускаются при открытии Serial-порта. В программе после подключения есть небольшая пауза, чтобы плата успела стартовать.

### Робот не реагирует

Проверьте:

- выбран правильный COM-порт;
- скорость 9600;
- Arduino прошит скетчем `RobotX_Nano_UART_Control.ino`;
- в Arduino IDE закрыт Serial Monitor, иначе порт может быть занят.
